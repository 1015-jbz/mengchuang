# 基于LoongArch端侧AI的智能座舱多模态交互终端 — 总体设计方案

## 一、项目概述

### 1.1 项目背景

随着智能网联汽车的快速发展，智能座舱已成为汽车"新四化"（电动化、智能化、网联化、共享化）战略的核心载体。当前主流智能座舱方案高度依赖云端AI服务，存在三大痛点：

- **网络依赖性强**：隧道、山区等无网环境功能降级甚至失效
- **隐私安全风险**：车内语音/视频数据上传云端，用户隐私难以保障
- **响应延迟不可控**：云端往返延迟影响安全预警的实时性

本项目基于**龙芯LoongArch架构**的端侧AI计算能力，构建全离线、低延迟、高隐私的智能座舱多模态交互终端。核心创新点在于将**语音交互、视觉感知、大模型推理**全部部署在车规级龙芯终端上，实现真正意义上的"端侧智能"。

### 1.2 项目目标

| 目标维度 | 具体指标 |
|---------|---------|
| 语音唤醒率 | ≥95%（车载噪声环境） |
| 语音识别准确率 | ≥93%（端侧离线ASR） |
| 疲劳检测准确率 | ≥90%（多模态融合） |
| 意图识别准确率 | ≥88%（本地NLU） |
| 语音响应延迟 | ≤500ms（端到端） |
| 危险预警延迟 | ≤200ms（视觉+传感器） |
| 大模型推理速度 | ≥15 tokens/s（7B量化模型） |

## 二、软件功能模块详解

### 2.1 模块总览

本项目共设计 **7大核心软件模块**，覆盖智能座舱的完整交互链路：

| 编号 | 模块名称 | 核心功能 | Python包名 |
|------|---------|---------|-----------|
| M1 | 音频前端处理 | 语音唤醒、波束成形、降噪增强 | `asr` |
| M2 | 语音识别引擎 (ASR) | 流式语音转文字、多语种支持 | `asr` |
| M3 | 意图理解引擎 (NLU) | 意图分类、槽位填充、多轮对话 | `nlu` |
| M4 | 端侧大模型推理 (LLM) | 量化推理、RAG知识库、流式输出 | `llm` |
| M5 | 驾驶安全监控 | 疲劳检测、分心检测、危险行为预警 | `safety` |
| M6 | 情感计算引擎 | 驾驶员情绪识别、个性化情感响应 | `emotion` |
| M7 | 座舱控制中间件 | 自然语言车控、场景联动、状态管理 | `control` |

### 2.2 M1+M2 — 语音交互模块

#### 功能列表

| 功能编号 | 功能名称 | 功能描述 |
|---------|---------|---------|
| F1.1 | 语音唤醒 (Keyword Spotting) | 支持自定义唤醒词（默认"你好小航"），低功耗常驻运行 |
| F1.2 | 麦克风阵列处理 | 多麦波束成形、自适应降噪、回声消除(AEC) |
| F1.3 | 语音活动检测 (VAD) | 实时检测语音起止，支持长句自动断句 |
| F1.4 | 流式语音识别 | 基于Whisper/WeNet的端侧流式ASR，支持中英文混合识别 |
| F1.5 | 声纹识别 | 区分主驾/副驾说话人，支持多用户个性化 |
| F1.6 | 语音合成 (TTS) | 自然情感语音合成，支持多种音色 |

#### 技术方案

```
音频输入 → 唤醒词检测 → VAD断句 → 特征提取 → ASR解码 → 文本输出
    │                      │
    └── 波束成形+降噪 ──────┘
```

- **唤醒词检测**: 基于轻量级CNN模型（~500KB），实时率<0.3
- **ASR引擎**: Whisper-small量化模型（~300MB），或WeNet流式模型
- **VAD**: 基于能量+神经网络的混合VAD方案

### 2.4 M4 — 端侧大模型推理模块

#### 功能列表

| 功能编号 | 功能名称 | 功能描述 |
|---------|---------|---------|
| F3.1 | 模型量化加载 | 支持INT4/INT8量化，7B模型内存占用<5GB |
| F3.2 | 流式文本生成 | Token级别流式输出，降低首字延迟 |
| F3.3 | RAG知识库 | 车载手册、常见问题等结构化知识的检索增强 |
| F3.4 | 多轮对话记忆 | 滑动窗口+摘要压缩的长期记忆管理 |
| F3.5 | 安全护栏 | 敏感话题过滤、有害内容拦截、幻觉检测 |
| F3.6 | 提示词工程 | 驾驶场景专用System Prompt，角色扮演一致性 |

#### 推荐模型方案

| 模型 | 参数量 | 量化方案 | 内存占用 | 适用场景 |
|------|--------|---------|---------|---------|
| Qwen2.5-7B-Instruct | 7B | INT4 GPTQ | ~4.5GB | 通用对话、意图解析 |
| Qwen2.5-1.5B-Instruct | 1.5B | INT8 | ~1.8GB | 轻量任务、快速响应 |
| ChatGLM3-6B | 6B | INT4 | ~4GB | 中文优化对话 |
| InternLM2-7B | 7B | INT4 | ~4.5GB | 工具调用、指令跟随 |
| BGE-M3 (Embedding) | 568M | FP16 | ~1.2GB | RAG检索向量化 |

> **LoongArch适配**: 上述模型需编译LoongArch版的推理引擎（llama.cpp / ONNX Runtime），或使用龙芯官方适配的推理框架。

### 2.6 M6 — 情感计算与陪伴模块

#### 功能列表

| 功能编号 | 功能名称 | 功能描述 |
|---------|---------|---------|
| F5.1 | 多模态情绪识别 | 融合面部表情(ViT)、语音语调(Wav2Vec2)、文本情感(BERT)三维度情绪分析 |
| F5.2 | 情绪画像构建 | 长期追踪用户情绪状态，建立个性化情感基线 |
| F5.3 | 主动情感关怀 | 检测到负面情绪时主动发起关怀对话 |
| F5.4 | 共情对话生成 | LLM生成具有共情能力的自然语言回应 |
| F5.5 | 情绪调节推荐 | 根据情绪状态推荐音乐/氛围灯/香氛/座椅按摩 |
| F5.6 | 驾驶信心增强 | 紧张/焦虑驾驶时提供鼓励和引导 |
| F5.7 | 亲子/伴侣模式 | 识别车内关系，调整交互风格 |

#### 情绪维度模型 (Plutchik's Wheel + 驾驶场景适配)

```
正面情绪                    负面情绪
──────────────────────────────────────
😊 愉悦 ─────────────── 😤 愤怒（路怒检测）
😌 平静 ─────────────── 😰 焦虑（新手/复杂路况）
🤩 兴奋 ─────────────── 😔 悲伤（情绪低落）
😎 自信 ─────────────── 😴 疲倦（疲劳预警）
                          😱 恐惧（紧急避险）
```

#### 情感陪伴场景示例

| 场景 | 检测信号 | 系统响应 |
|------|---------|---------|
| 路怒倾向 | 语速加快+音量升高+频繁急刹 | "深呼吸，我们不着急，安全最重要" + 舒缓音乐 |
| 新手紧张 | 心率升高+方向盘微调频繁+表情紧绷 | "你开得很好，前面200米右转，我提前提醒你" |
| 孤独夜驾 | 长时间无对话+深夜时段+单调路况 | "今晚的星空很美呢，要不要听首轻松的歌？" |
| 疲劳驾驶 | PERCLOS>0.3+打哈欠+车道偏离 | "你已经连续驾驶2小时了，前方3公里有服务区，休息一下吧" |
| 情绪低落 | 语音低沉+表情悲伤+沉默 | "我一直在呢，如果你想聊聊，我很愿意听" |

## 五、关键技术难点与解决方案

| 难点 | 描述 | 解决方案 |
|------|------|---------|
| 端侧算力限制 | 7B模型推理对CPU/内存要求高 | INT4量化+llama.cpp优化+NPU卸载 |
| 车载噪声 | 行驶中风噪/胎噪/音乐干扰ASR | 麦克风阵列+频域降噪+车载声学模型微调 |
| LoongArch适配 | 很多AI框架没有LoongArch预编译包 | 源码编译+龙芯社区支持+容器化部署 |
| 实时性要求 | 安全预警需要极低延迟 | 多线程流水线+模型级联（轻量模型初筛→大模型精判） |
| 情感理解主观性 | 情绪识别存在误判 | 多模态融合提高准确率+用户反馈闭环 |
